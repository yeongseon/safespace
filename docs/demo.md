# 🎮 데모 시나리오

발표 중 짧은 시간 안에 SafeSpace의 가치를 명확히 보여주기 위한 시나리오 설계이다. 백엔드의 `DemoSimulator`와 프론트엔드의 `simulator.ts`에 동일한 로직으로 구현되어 있다.

---

## 📋 시나리오 개요

| #️⃣ # | 🎮 시나리오 | ⚠️ 위험 요소 | ⏱️ 진행 시간 | 🚨 최종 상태 |
|---|----------|-----------|-----------|-----------|
| 1 | `safe` | 없음 | 즉시 | 🟢 SAFE |
| 2 | `oxygen_drop` | 산소 감소 | 30초 | 🔴 CRITICAL |
| 3 | `gas_leak` | 가스 누출 | 18초 | 🔴 CRITICAL |
| 4 | `worker_collapse` | 작업자 쓰러짐 | 즉시 | 🔴 CRITICAL |
| 5 | `multi_risk` | 복합 위험 | 30초 | 🔴 CRITICAL |

!!! warning "적용 범위"
    모든 시나리오는 **`paint-tank-a` 구역에만** 적용된다. `cargo-hold-b`와 `engine-room-c`는 항상 safe 베이스라인을 유지한다.

---

## ✅ Scenario 1: Safe Baseline

모든 센서가 정상 범위, 작업자 normal 상태이다.

### 📊 센서 베이스라인 (구역별)

| 📊 센서 | paint-tank-a | cargo-hold-b | engine-room-c |
|------|-------------|-------------|---------------|
| O₂ | 20.8% ± 0.1 | 20.8% ± 0.1 | 20.8% ± 0.1 |
| H₂S | 1.0 ± 0.3 ppm | 1.0 ± 0.3 ppm | 1.0 ± 0.3 ppm |
| CO | 5.0 ± 1.4 ppm | 5.0 ± 1.4 ppm | 5.0 ± 1.4 ppm |
| VOC | 80 ± 10 ppm | 80 ± 10 ppm | 80 ± 10 ppm |
| 온도 | 25.8°C ± 1 | 22.6°C ± 1 | 29.5°C ± 1 |
| 습도 | 62% ± 3 | 65% ± 3 | 52% ± 3 |

### 🖥️ UI 반응

- 🔴 OverallRiskCard: 0~5점, 녹색 게이지
- 📊 모든 센서 카드: 녹색 뱃지
- 🚨 AlertBanner: 숨김
- 📝 이벤트 로그: 신규 이벤트 없음

---

## 🫁 Scenario 2: Oxygen Drop

30초에 걸쳐 산소가 점진적으로 감소한다. 환기 고장이나 산소 소비 시나리오를 재현한다.

### ⏱️ 진행 타임라인

| ⏱️ 경과 시간 | 🫁 O₂ 농도 | 🚨 예상 상태 |
|-----------|---------|-----------|
| 0초 | ~20.8% | 🟢 SAFE |
| 5초 | ~20.1% | 🟢 SAFE |
| 10초 | ~19.4% | 🟡 CAUTION |
| 15초 | ~18.7% | 🟠 WARNING |
| 20초 | ~17.9% | 🟠 WARNING ~ 🔴 CRITICAL |
| 25초 | ~17.2% | 🔴 CRITICAL |
| 30초 | ~16.5% | 🔴 CRITICAL |

**산소 공식**: `O₂ = 20.8 - 4.3 × min(elapsed/30, 1.0) + random(-0.08, 0.03)`

### 🖥️ UI 반응 순서

1. **0~8초**: 센서 카드 숫자가 서서히 감소, 게이지 미세 변화
2. **~10초**: CAUTION 전환 — 게이지 노란색, 센서 카드 노란 뱃지
3. **~15초**: WARNING 전환 — AlertBanner 슬라이드 다운 (주황), 이벤트 생성
4. **~20초**: CRITICAL 전환 — AlertBanner 빨간색 + 펄스, 조치 가이드 "즉시 대피"

---

## ☠️ Scenario 3: Gas Leak

18초에 걸쳐 H₂S, VOC, CO가 급상승한다. 가스 배관 파열이나 도장 작업 사고를 재현한다.

### ⏱️ 진행 타임라인

| ⏱️ 경과 시간 | ☠️ H₂S (ppm) | ☠️ VOC (ppm) | ☠️ CO (ppm) | 🚨 예상 상태 |
|-----------|-----------|-----------|----------|-----------|
| 0초 | ~2 | ~80 | ~7 | 🟢 SAFE |
| 4초 | ~7 | ~140 | ~13 | 🟡 CAUTION |
| 8초 | ~12 | ~200 | ~18 | 🟠 WARNING |
| 12초 | ~17 | ~260 | ~24 | 🟠 WARNING ~ 🔴 |
| 18초 | ~25 | ~350 | ~32 | 🔴 CRITICAL |

**가스 공식**:

- `H₂S = 2 + 23 × min(elapsed/18, 1.0) + random(-0.4, 0.6)`
- `VOC = 80 + 270 × min(elapsed/18, 1.0) + random(-8, 10)`
- `CO = 7 + 25 × min(elapsed/18, 1.0) + random(-1.5, 2)`

### 🖥️ UI 반응

- 📊 여러 센서 카드가 동시에 경고색으로 전환
- 📝 이벤트 로그에 "elevated H2S", "elevated CO", "elevated VOC" 메시지

---

## 🎥 Scenario 4: Worker Collapse

작업자가 즉시 `fall_suspected` 상태로 전환된다. 영상 기반 이상행동 감지를 재현한다.

### 🔄 상태 변화

| 📋 항목 | 📥 값 |
|------|-----|
| worker_status | `fall_suspected` |
| confidence | 0.96 |
| last_motion_seconds | max(12, elapsed × 1.4) — 시간이 지남에 따라 증가 |

센서 데이터는 변화 없음 (safe 베이스라인 유지).

### 🖥️ UI 반응

- **VideoMonitorPanel**: 빨간 오버레이, "쓰러짐 의심 (96%)" 표시
- **WorkerStatusCard**: 빨간 뱃지, fall_suspected
- **OverallRiskCard**: 작업자 위험도 90 × 0.25 = 22.5점 반영
- 센서만으로는 SAFE이지만, 작업자 상태로 인해 CAUTION 수준

---

## 🚨 Scenario 5: Multi-Risk

산소 감소 + 가스 누출 + 작업자 쓰러짐이 동시에 발생한다. 최악의 상황을 재현한다.

### ⚙️ 복합 진행

| 📋 요소 | ⏱️ 진행 시간 | 🧾 공식 |
|------|-----------|------|
| 산소 감소 | 30초 | oxygen_drop과 동일 |
| 가스 누출 | 20초 | gas_leak과 유사 (진행 속도 약간 다름) |
| 작업자 | 즉시 | fall_suspected, confidence=0.98, last_motion=max(15, elapsed×1.6) |

### 📊 예상 점수 (30초 후)

```
oxygen:  ~93.75 × 0.35 = 32.8
gas:     ~100   × 0.30 = 30.0
env:     ~0     × 0.10 = 0.0
worker:  90     × 0.25 = 22.5
rapid:   ~100   × 0.15 = 15.0
────────────────────────
합계: ~100점 → CRITICAL
```

### 🖥️ UI 반응

- **모든 요소가 CRITICAL 반응**: 빨간 게이지, 빨간 배너, 빨간 비디오 오버레이
- 이벤트 로그에 다수 이벤트 동시 생성
- 조치 가이드: "즉시 작업 중단. 전원 대피. 구조팀 투입."

## ⚙️ 클라이언트 vs 서버 시뮬레이터

| 📋 항목 | ⚙️ 백엔드 (`DemoSimulator`) | 🖥️ 프론트엔드 (`simulator.ts`) |
|------|------------------------|---------------------------|
| 실행 환경 | Python (서버) | TypeScript (브라우저) |
| 데이터 저장 | SQLite DB | Zustand Store (메모리) |
| 통신 방식 | WebSocket broadcast | 직접 스토어 갱신 |
| 시나리오 공식 | 동일 | 동일 (포팅) |
| 위험 평가 | `rules.py` | `evaluateRisk()` |
| 구역 오프셋 | 적용 | 미적용 (paint-tank-a 기준) |
| 사용 시나리오 | 로컬 개발, Full-stack | GitHub Pages 정적 배포 |

## 🎤 권장 발표 순서

| #️⃣ 순서 | 🎮 시나리오 | ⏱️ 시간 | 🎤 발표 포인트 |
|------|----------|------|-------------|
| 1 | `safe` | 1분 | 시스템 소개, UI 구성 설명, 실시간 갱신 시연 |
| 2 | `oxygen_drop` | 1분 | 점진적 위험 감지, 상태 전환 과정 시연 |
| 3 | `gas_leak` | 1분 | 복수 센서 동시 경고, 급변 감지 설명 |
| 4 | `worker_collapse` | 1분 | 영상 기반 감지, 복합 위험 판정 설명 |
| 5 | `multi_risk` | 1분 | 최악의 상황 대응, 모든 UI 요소 CRITICAL 반응 |
| 6 | (발표) | 1분 | 기대효과, 확장 방향, Q&A |

!!! tip "시나리오 전환 팁"
    시나리오 사이에 `safe`를 실행하여 초기 상태로 복귀한 후 다음 시나리오를 시작하면 전환 과정이 더 명확하게 보인다.
