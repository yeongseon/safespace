    제품 요구사항 (PRD)¶

    SafeSpace의 제품 요구사항을 정의하는 문서이다.

    Product Overview¶

    항목 내용
    제품명 SafeSpace
    제품 유형 웹 기반 산업 안전 모니터링 대시보드
    비전 실시간 환경 센싱과 작업자 행동 분석을 결합한 밀폐공간 예방적 안전 모니터링 시스템
    목표 환경 조선소 내 밀폐공간 (도장 탱크, 화물창, 기관실 등)

    문제 정의¶

    밀폐공간 작업은 산소 결핍, 독성 가스 축적, 작업자 이상 인지 지연으로 인해 급속히 치명적 상황으로 발전할 수 있다. 기존 안전 관행은 수동 점검과 사람의 관찰에 의존하므로 빠른 사고 예방에는 한계가 있다.

    사고 특성

    밀폐공간 질식사고는 발생 후 수 분 내에 치명적 결과로 이어진다. 또한 구조자가 상황을 인지하지 못하고 진입하여 2차 사고가 발생하는 경우가 빈번하다.

    대상 사용자¶

    페르소나 1: 현장 안전관리자¶

    항목 내용
    역할 밀폐공간 작업 감독 및 비상 대응 총괄
    핵심 니즈 여러 구역의 상태를 한 화면에서 파악, 위험 발생 시 즉시 인지
    Pain Point 현재는 무전기와 수동 점검에 의존하여 인지 지연 발생
    기대 위험 감지 시 자동 경고와 조치 가이드 수신

    페르소나 2: 작업구역 관리자¶

    항목 내용
    역할 특정 구역의 작업 상태 및 환경 모니터링
    핵심 니즈 담당 구역의 센서 추이를 실시간으로 확인
    Pain Point 정기 점검 주기 사이에 상태 변화를 놓침
    기대 센서 차트와 이벤트 이력으로 패턴 파악

    페르소나 3: 공모전 심사위원 / 발표 청중¶

    항목 내용
    역할 서비스 가치 평가
    핵심 니즈 짧은 시간 내 시스템 가치와 기술적 완성도 확인
    기대 직관적인 UI와 실감나는 데모 시나리오

    MVP 범위¶

    In Scope¶

    • 실시간 센서 데이터 표시 (O₂, H₂S, CO, VOC, 온도, 습도)
    • 가중 점수제 기반 4단계 위험 등급 산정
    • 5분 이동평균 대비 급변 감지
    • 이벤트 로그 및 관리자용 조치 가이드
    • 작업자 상태 모니터링 (정상, 무동작, 쓰러짐)
    • 5가지 데모 시나리오 시뮬레이션
    • WebSocket 기반 실시간 브로드캐스트
    • GitHub Pages 정적 데모 배포
    • MkDocs 기반 문서화

    Out of Scope¶

    • 실제 산업용 인증 하드웨어 (IoT 센서) 연동
    • 웨어러블 디바이스 연동
    • 다중 사업장 통합 관제
    • 모바일 네이티브 앱
    • 고도화된 예측 AI 모델 (LSTM, Transformer 등)
    • 자동 환기 제어 연동
    • 사용자 인증 / 권한 관리

    핵심 사용 사례¶

    UC-01: 실시간 작업구역 상태 확인¶

    항목 내용
    트리거 관리자가 대시보드 접속
    주요 흐름 대시보드에서 전체 위험도 게이지, 센서 카드, 작업자 상태, 시계열 차트 확인
    성공 기준 접속 후 3초 이내에 전체 상태를 파악할 수 있다

    UC-02: 위험 발생 감지 및 경고¶

    항목 내용
    트리거 센서 임계치 초과 또는 급변 감지
    주요 흐름 위험 점수 재산정 → 상태 전환 → AlertBanner 표시 → 이벤트 로그 생성 → 조치 가이드 갱신
    성공 기준 WARNING/CRITICAL 전환이 2초 이내에 UI에 반영된다

    UC-03: 작업자 이상행동 감지¶

    항목 내용
    트리거 작업자 쓰러짐 또는 장시간 무동작
    주요 흐름 작업자 상태 변경 → 비디오 모니터 빨간 오버레이 → 이벤트 생성 → 위험 점수 반영
    성공 기준 fall_suspected 시 비디오 패널에 즉시 경고 표시

    UC-04: 발표용 시뮬레이션¶

    항목 내용
    트리거 Demo 페이지에서 시나리오 버튼 클릭
    주요 흐름 시나리오 활성화 → 센서값 점진적 변화 → 위험도 전환 → 전체 UI 반응
    성공 기준 버튼 하나로 실감나는 시연이 가능하다

    기능 요구사항¶

    ID 기능 요구사항 우선순위
    FR-01 센서 모니터링 6종 센서 데이터를 2초 간격으로 갱신, 최근 300개 데이터 포인트 차트 표시 P0
    FR-02 위험 평가 가중 점수제(O₂ 35%, 가스 30%, 환경 10%, 작업자 25%) + 급변 감지(15% 가산) P0
    FR-03 이벤트 로그 시간순 이벤트 기록 (유형, 심각도, 시각, 구역, 설명, 출처) P0
    FR-04 작업자 모니터링 3가지 상태(normal, inactive, fall_suspected) 표시, confidence 수치 P0
    FR-05 경고 시스템 WARNING 시 노란 배너, CRITICAL 시 빨간 배너 + 펄스 애니메이션 P0
    FR-06 데모 컨트롤 5가지 시나리오 버튼으로 위험 상태 재현 P0
    FR-07 구역 관리 3개 기본 구역(도장탱크, 화물창, 기관실) 상태 표시 P1
    FR-08 조치 가이드 위험 단계별 즉시 조치 권고 메시지 표시 P1
    FR-09 실시간 통신 WebSocket 기반 양방향 통신, 연결 상태 표시기 P0
    FR-10 정적 배포 클라이언트 사이드 시뮬레이터로 백엔드 없이 완전 동작 P0

    비기능 요구사항¶

    ID 항목 요구사항 측정 기준
    NFR-01 사용성 첫 화면에서 3초 이내 전체 상태 파악 가능 사용자 테스트
    NFR-02 성능 2초 간격 갱신, 부드러운 애니메이션 (60fps) Chrome DevTools
    NFR-03 안정성 데모 시나리오 반복 실행 시 UI 깨짐 없음 5회 연속 시나리오 전환
    NFR-04 시각 품질 다크 테마 산업 관제 UI, 일관된 색상 시스템 디자인 리뷰
    NFR-05 호환성 Chrome, Firefox, Safari 최신 버전 지원 브라우저 테스트
    NFR-06 배포 용이성 GitHub Pages에서 별도 서버 없이 동작 정적 빌드 확인

    성공 지표¶

    • 정상/주의/위험/치명 상태 전환이 색상과 애니메이션으로 명확하게 구분된다
    • 5개 시나리오를 발표 중 안정적으로 재현 가능하다
    • 센서 차트, 이벤트 로그, 경고 배너, 조치 가이드가 하나의 흐름으로 연결된다
    • 심사위원이 "현업 적용 가능성"을 이해할 수 있다
    • GitHub Pages 데모 URL만으로 별도 설치 없이 체험 가능하다

    용어 정의¶

    용어 정의
    밀폐공간 외부와 격리되어 환기가 제한되는 작업 공간 (탱크, 맨홀, 배관 내부 등)
    O₂ (산소) 정상 대기 농도 약 20.9%. 18% 미만 시 질식 위험
    H₂S (황화수소) 무색 유독가스. 10ppm 이상 시 건강 위험. 부패한 달걀 냄새
    CO (일산화탄소) 무색무취 유독가스. 50ppm 이상 시 두통, 어지럼증
    VOC (휘발성유기화합물) 도장 작업 시 발생하는 유해 증기. 200ppm 이상 시 위험
    Risk Score 0~100 범위의 복합 위험도 점수. 가중 점수제로 산정
    fall_suspected 작업자 쓰러짐 의심 상태. 영상 분석 또는 시뮬레이션으로 감지
