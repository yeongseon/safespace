# 🎤 발표 가이드

SafeSpace 프로젝트 발표를 위한 스크립트, 차별화 포인트, Q&A 대비 자료이다.

## 📌 핵심 메시지

!!! note "한 줄 요약"
    본 시스템은 밀폐공간 질식사고를 **사후 대응**이 아닌 **사전 예방** 관점에서 접근한다. 실시간 센서 데이터와 영상 분석을 결합하여 **사고 직전 징후를 감지**하고 즉시 경고한다.

## 🎤 6분 발표 스크립트

### ⚠️ 1분: 문제 제기

> "조선소 밀폐공간에서 질식사고가 발생하면, 수 분 내에 치명적 결과로 이어집니다. 더 심각한 것은 구조자가 상황을 모르고 진입하여 2차 사고가 발생하는 경우가 빈번하다는 것입니다."
>
> "기존에는 작업 전 가스 측정기로 한 번 확인하고 들어갑니다. 하지만 작업 중 환경이 변하면 아무도 모릅니다."

**→ `safe` 시나리오 실행. 정상 상태의 대시보드를 보여주며 시스템 구성 설명.**

### 🫁 1분: 산소 감소 시연

> "지금부터 환기가 중단된 상황을 시뮬레이션합니다. 산소가 서서히 떨어지는 것을 보세요."

**→ `oxygen_drop` 시나리오 실행. 30초간 SAFE→CAUTION→WARNING→CRITICAL 전환 시연.**

> "10초 만에 주의 단계, 15초에 경고 배너가 뜹니다. 기존 방식이라면 이 시점에서는 아무도 모르고 작업을 계속하고 있을 겁니다."

### ☠️ 1분: 가스 누출 시연

> "이번에는 가스 배관 누출 상황입니다. H₂S, VOC, CO가 동시에 급상승합니다."

**→ `gas_leak` 시나리오 실행.**

> "주목할 점은 '급변 감지' 기능입니다. 절대값이 아직 위험 범위가 아니어도, 변화 속도가 빠르면 미리 경고합니다."

### 🎥 1분: 작업자 쓰러짐

> "환경 센서만으로는 감지할 수 없는 상황도 있습니다. 작업자가 갑자기 쓰러지면?"

**→ `worker_collapse` 시나리오 실행. 비디오 패널 빨간 오버레이 주목.**

> "영상 분석으로 쓰러짐을 감지하고, 96% 신뢰도로 경고합니다. 센서는 정상이지만 작업자 위험도가 복합 점수에 반영됩니다."

### 🚨 1분: 최악의 상황

> "마지막으로, 모든 것이 동시에 발생하는 최악의 시나리오입니다."

**→ `multi_risk` 시나리오 실행. 모든 UI 요소가 CRITICAL로 전환되는 모습.**

> "산소 감소, 가스 누출, 작업자 쓰러짐이 동시에 발생. 시스템은 즉시 '전원 대피, 구조팀 투입'을 권고합니다."

### ✅ 1분: 마무리

> "SafeSpace는 '사고가 난 다음에 어떻게 할 것인가'가 아니라 '사고가 나기 직전에 어떻게 감지할 것인가'에 대한 답입니다."
>
> "현재 MVP는 시뮬레이션 기반이지만, 실제 IoT 센서와 MediaPipe 영상 분석을 연동하면 현업에 바로 적용할 수 있는 구조입니다. GitHub Pages에서 바로 체험해 보실 수 있습니다."

## ❓ Q&A 대비

### 📊 Q: 실제 센서 없이 어떻게 동작하나요?

> "MVP는 시뮬레이터로 센서 데이터를 생성합니다. 하지만 백엔드의 센서 입력 인터페이스는 REST API로 설계되어 있어, 실제 IoT 센서가 HTTP POST로 데이터를 보내면 바로 연동됩니다. 시뮬레이터를 실제 센서로 교체하는 것은 API 엔드포인트 하나의 변경입니다."

### ⚠️ Q: 위험 판정의 근거는?

> "산업안전보건법 및 KOSHA GUIDE의 밀폐공간 관리 기준을 참고했습니다. 산소 18% 미만, H₂S 10ppm 초과 등의 임계치는 실제 규정 값입니다. 가중치는 산소(35%)를 최우선으로, 질식이 가장 즉각적인 위험이기 때문입니다."

### 🎥 Q: 영상 분석의 정확도는?

> "현재 MVP는 시뮬레이션입니다. 실제 구현 시 MediaPipe Pose Estimation을 사용하면 쓰러짐 감지 정확도 90% 이상이 보고되고 있습니다. MVP의 구조가 이를 바로 수용할 수 있도록 설계되어 있습니다."

### 🛡️ Q: 기존 안전 시스템과의 차이점은?

> "기존 시스템은 '측정'에 집중합니다. 가스 감지기가 울리면 이미 위험한 상태입니다. SafeSpace는 '예측'에 집중합니다. 급변 감지 기능으로 아직 안전 범위이더라도 빠르게 나빠지고 있다면 미리 경고합니다."

### 🗺️ Q: 확장성은?

> "백엔드는 FastAPI + SQLModel로 구축되어, PostgreSQL로 전환하고 여러 사업장의 구역을 추가하는 것이 용이합니다. 프론트엔드의 Zone 시스템도 동적으로 구역을 추가/제거할 수 있는 구조입니다."

### 🌐 Q: 왜 웹 기반인가요?

> "관제실에서는 다양한 OS와 환경이 혼재합니다. 웹 기반이면 브라우저만 있으면 되고, 설치가 필요 없습니다. 또한 GitHub Pages에 데모를 올려서 누구나 URL 하나로 바로 체험할 수 있습니다."

## ⚠️ 데모 트러블슈팅

| ❗ 문제 | 🔍 원인 | ✅ 해결 |
|------|------|------|
| 데모 페이지가 안 열림 | GitHub Pages 배포 지연 | 1~2분 대기, 또는 로컬에서 `npm run dev` |
| 시나리오 반응 없음 | 시뮬레이터 미시작 | 페이지 새로고침 (startSimulator 재실행) |
| 차트가 비어있음 | 데이터 누적 전 | safe 상태로 10~20초 대기 후 시나리오 실행 |
| 게이지가 0에서 안 움직임 | 브라우저 탭이 비활성 | setInterval이 throttle됨. 탭 활성화 |
